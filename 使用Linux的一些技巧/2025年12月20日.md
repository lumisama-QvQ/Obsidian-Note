# 彻底解决sway下缩放问题
sway不支持单独为XWayland设置DPI，这会导致X11应用模糊，在`sway`下唯一能较好的解决的办法便是关闭`sway`的DPI设置，将全局设置为`1`，之后单独为应用设置DPI。
```config
# ~/.Xresources
//缩放1.5
Xft.dpi: 144
```

```bash
#~/.profile
export GDK_SCALE=1
export GDK_DPI_SCALE=1.5
export QT_AUTO_SCREEN_SCALE_FACTOR=0
export QT_SCALE_FACTOR=1.5
```
对于QQ：
在`.config`文件夹下新建`qq-flags.conf`文件
添加参数：
```bash
# 设置缩放1.5
--force-device-scale-factor=1.5
```
此参数其他的`election`框架软件也可使用，效果一致
# Linux下使用stow管理设置
我们可以通过`stow`快速管理和整合配置文件

首先在你的主目录下新建一个`.dotfiles`的文件夹作为管理库，随后在其中为你想要的软件建立单独的文件夹。
这里比如我要管理`sway`的设置`~/.config/sway`，我们在`.dotfiles`中新建：
```bash
mkdir -p sway/.config/
```
随后将原配置文件夹整个移入我们创建的文件夹里：
```bash
mv ~/.config/sway sway/.config/
```
之后在`.dotfiles`文件夹中执行`stow sway`即可在原配置的位置创建链接，在`sway`眼中，配置还在那，但实际上已经集中管理了。
之后我们就可以通过云备份这个文件夹，实现配置的云端备份。

不只是`.config/`下的配置可以这么做，注意到我们为什么要在`sway`的配置文件夹下再建一个`.config/`了吗？

实际上`stow`会以每一个软件的配置目录当作是用户的主目录，并根据其中的内容
